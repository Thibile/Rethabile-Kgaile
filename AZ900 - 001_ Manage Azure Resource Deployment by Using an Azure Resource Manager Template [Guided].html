Manage Azure Resource Deployment by Using an Azure Resource Manager Template


Challenge Overview

Understand the scenario

You are an Azure® administrator responsible for your organization’s Azure environment. You need to deploy resources by using Azure Resource Manager (ARM) templates. First, you will export an ARM template by using the Azure portal. Next, you will use a custom template to deploy a virtual machine. Finally, you will display the Azure API by using the Azure Resource Explorer.

Understand your environment

You will be using a pre-created Azure account that contains a resource group named AZ900RGlod12811506 and a storage account named sa12811506.

    Azure Resource Manager (ARM) is responsible for managing deployment and management services for Azure. ARM gives you a consistent management layer for creating, updating, and deleting resources, and it supports the use of templates for automated deployments.

    ARM templates are JavaScript Object Notation (JSON) files that you can create by using the Azure portal or any JSON editor—for example, Visual Studio Code. You can create a template from scratch, or you can edit an existing Azure Quickstart template, and then save it as a new template.

    After you create a template, you can deploy it to Azure by using the Azure portal, Visual Studio, the Azure PowerShell® New-AzResourceGroupDeployment cmdlet, and the command-line interface (CLI) 2.0 az group deployment create command.

    You can store templates on a local computer or in a repository. Once downloaded, you can edit the template and use it to deploy a new Azure resource.

    • Review the information on the Very simple deployment of a Windows VM page, and then select Deploy to Azure.

    • Verify that the Custom deployment page is authenticated to your Admin-12811506@cloudslice.onmicrosoft.com account.

    The ARM template used in this challenge is stored in a GitHub repository.

    Virtual machines are one of the most frequently deployed resources in Azure. When your company decides to migrate resources to the cloud, virtual machines can, and often do, replace on-premises servers. Virtual machines can be generic, containing only an operating system, or they can contain types of servers—for example, SQL Server, Exchange, and SharePoint. You can use pre-created Quickstart images or you can create custom images.

    To deploy a virtual machine, you can use the Azure portal, Azure PowerShell cmdlets, CLI 2.0 commands, or ARM templates. Visual Studio and Visual Studio Code are useful for deploying resources by using ARM templates.

    You should size your Azure virtual machines appropriately to achieve a balance between the desired level of performance and your cost-management goals.

    When you create a virtual machine, you can use an existing virtual network, or you can create a new virtual network during the deployment process.

Display the Azure API by using the Azure Resource Explorer

  • Open the Azure Resource Explorer by using the Azure portal.

    Expand this hint for guidance on opening the Azure Resource Explorer.

    • On the Azure portal menu, select All Services.

    • Search for Resource, and then select Resource Explorer.

    You can use Azure Resource Explorer to learn the Azure API, and to determine which resource providers are being used by your Azure account. You can access the Resource Explorer directly from the Azure portal or at https://resources.azure.com.

    more...

  • Review the list of resource providers that are available to the Azure platform.

    Expand this hint for guidance on reviewing the list of resource providers.

    • On the Resource Explorer menu, expand Providers, and then review the list of providers.

  • Review the list of resource providers that are being used by your subscription.

    Expand this hint for guidance on reviewing the list of resource providers that are being used by your subscription.

    • On the Resource Explorer menu, expand Subscriptions, expand Providers, and then view the list of providers that are being used by your subscription.

  • Review the details of the resources that are deployed to the AZ900RGlod12811506 resource group.

    Expand this hint for guidance on reviewing the details of the resources in a resource group.

    • On the Resource Explorer menu, expand ResourceGroups, expand AZ900RGlod12811506, expand Resources, and then select each resource to view the details provided.
